快速入門¶

本頁提供此程式庫的簡要介紹。它假設您已經安裝了程式庫，如果尚未安裝，請參閱 安裝 章節。

最小化機器人¶

讓我們建立一個會回應特定訊息的機器人，並逐步帶您了解。

它看起來像這樣：

# 此範例需要 'message_content' 意圖。

import discord

intents = discord.Intents.default()
intents.message_content = True

client = discord.Client(intents=intents)

@client.event
async def on_ready():
    print(f'我們已以 {client.user} 身份登入')

@client.event
async def on_message(message):
    if message.author == client.user:
        return

    if message.content.startswith('$hello'):
        await message.channel.send('Hello!')

client.run('在此輸入您的 token')

讓我們將此檔案命名為 example_bot.py。請確保不要將其命名為 discord.py，因為這會與程式庫產生衝突。

這裡有很多內容，讓我們逐步帶您了解。

  1. 第一行只是匯入程式庫，如果這引發了 ModuleNotFoundError 或 ImportError，請前往 安裝 章節進行正確安裝。

  2. 接下來，我們建立一個 Client 的實例。此 client 是我們與 Discord 的連線。

  3. 然後我們使用 Client.event() 裝飾器來註冊事件。此程式庫有許多事件。由於此程式庫是異步的，我們以「回呼」（callback）風格的方式來處理事情。

    回呼本質上是一個在特定事件發生時被呼叫的函式。在我們的例子中，on_ready() 事件會在機器人完成登入與設定後被呼叫，而 on_message() 事件則在機器人收到訊息時被呼叫。

  4. 由於 on_message() 事件會對*每則*收到的訊息觸發，我們必須確保忽略來自自己的訊息。我們透過檢查 Message.author 是否與 Client.user 相同來做到這一點。

  5. 之後，我們檢查 Message.content 是否以 '$hello' 開頭。如果是，則在使用該指令的頻道中發送 'Hello!'。這是一種基本的指令處理方式，之後可以使用 discord.ext.commands – Bot commands framework 框架來自動化處理。

  6. 最後，我們使用登入 token 來執行機器人。如果您需要取得 token 或建立機器人的協助，請查閱 建立機器人帳號 章節。

現在我們已經建立了一個機器人，我們必須*執行*它。幸運的是，這很簡單，因為這只是一個 Python 腳本，我們可以直接執行它。

在 Windows 上：

$ py -3 example_bot.py

在其他系統上：

$ python3 example_bot.py

現在您可以試著玩玩看您的基本機器人了。
