版本保證¶

此程式庫遵循 語意化版本原則，這表示每當有不相容的 API 變更時，主要版本號就會更新。然而，由於 Discord 方面對破壞性變更缺乏保證，加上 Python 本身的動態特性，有時很難區分什麼算是破壞性變更，什麼不算。

首先要記住的是，破壞性變更僅適用於**公開文件中的函數與類別**。如果某個項目未列在此處的文件中，則它不屬於公開 API 的一部分，因此可能會變更。這包括以底線開頭的屬性，或是沒有底線但未記載於文件中的函數。

備註

以下範例並非詳盡列舉。

破壞性變更的範例¶

  • 將預設參數值更改為其他值。

  • 重新命名函數但未提供指向舊函數的別名。

  • 對事件新增或移除參數。

非破壞性變更的範例¶

  • 新增或移除私有底線屬性。

  • 在資料類別的 __slots__ 中新增元素。

  • 修改函數行為以修正錯誤。

  • Changes in the typing behaviour of the library

  • Changes in the calling convention of functions that are primarily meant as callbacks

  • 文件內容的變更。

  • 修改內部的 HTTP 處理方式。

  • 將依賴項目升級至新版本，無論是主要版本或其他版本。
