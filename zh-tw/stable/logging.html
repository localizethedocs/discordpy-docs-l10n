設定日誌記錄¶

discord.py 透過 logging Python 模組來記錄錯誤與除錯資訊。為了簡化此流程，程式庫在使用 Client.run() 時會為 discord 日誌記錄器提供預設設定。強烈建議您設定日誌模組，因為如果未進行設定，將不會輸出任何錯誤或警告。

程式庫提供的預設日誌設定會使用彩色輸出將內容列印到 sys.stderr。您也可以改用內建的 logging.handlers`（例如 :class:`logging.FileHandler）將其設定為輸出到檔案。

這可以透過將其傳遞給 Client.run() 來完成：

import logging

handler = logging.FileHandler(filename='discord.log', encoding='utf-8', mode='w')

# 假設 client 指向一個 discord.Client 子類別...
client.run(token, log_handler=handler)

您也可以透過傳遞 None 來完全停用程式庫的日誌設定：

client.run(token, log_handler=None)

同樣地，也可以將日誌層級設定為 logging.DEBUG：

import logging

handler = logging.FileHandler(filename='discord.log', encoding='utf-8', mode='w')

# 假設 client 指向一個 discord.Client 子類別...
client.run(token, log_handler=handler, log_level=logging.DEBUG)

建議這樣做，特別是在 DEBUG 等詳細層級下，因為會記錄大量事件，若輸出到 stderr 可能會淹沒您程式的終端輸出。

如果您希望程式庫提供的日誌設定影響所有日誌記錄器，而不僅是 discord 日誌記錄器，可以在 Client.run() 中傳遞 root_logger=True：

client.run(token, log_handler=handler, root_logger=True)

如果您想在不使用 Client.run() 的情況下使用程式庫提供的設定來設置日誌，可以使用 discord.utils.setup_logging()：

import discord

discord.utils.setup_logging()

# 或者，例如
discord.utils.setup_logging(level=logging.INFO, root=False)

使用 logging 模組可以進行更進階的設定。以下範例設定了一個循環檔案處理器，對程式庫輸出的所有內容（HTTP 請求除外）輸出 DEBUG 層級的記錄：

import discord
import logging
import logging.handlers

logger = logging.getLogger('discord')
logger.setLevel(logging.DEBUG)
logging.getLogger('discord.http').setLevel(logging.INFO)

handler = logging.handlers.RotatingFileHandler(
    filename='discord.log',
    encoding='utf-8',
    maxBytes=32 * 1024 * 1024,  # 32 MiB
    backupCount=5,  # 在 5 個檔案之間循環
)
dt_fmt = '%Y-%m-%d %H:%M:%S'
formatter = logging.Formatter('[{asctime}] [{levelname:<8}] {name}: {message}', dt_fmt, style='{')
handler.setFormatter(formatter)
logger.addHandler(handler)

# 假設 client 指向一個 discord.Client 子類別...
# 抑制預設設定，因為我們已有自己的設定
client.run(token, log_handler=None)

如需更多資訊，請查閱 logging 模組的文件與教學。

在 2.0 版本變更: 此程式庫現在提供預設的日誌設定。
